SGLang — April 2026

Three critical RCEs in one of the most widely deployed LLM inference frameworks.

All three vulnerabilities were confirmed exploitable and missed by CodeQL and Semgrep. Proof of concept available in the data room.

CVSS 10.0 SSTI via chat_template_kwargs

User-controlled input reaches a Jinja2 Template().render() call through apply_chat_template(). Unauthenticated remote code execution. Proof of concept available in the data room.

CVSS 8.1 dill.loads deserialization in CustomLogitProcessor

User-supplied logit processor configurations are deserialized with dill.loads() without validation. Allows arbitrary code execution via a crafted payload.

CVSS 10.0 ZMQ pickle regression — incomplete CVE-2026-3060 patch

CVE-2026-3060 was partially patched. Rowan detected the remaining code path where serialization was still reachable. Unauthenticated RCE restored after the incomplete patch.

Letta — May 2026

Two critical RCEs in the core tool executor.

Critical eval() on LLM-controlled strings

The CLI interface passes LLM output directly to eval(). Any LLM response can execute arbitrary Python in the scanner process.

Critical exec() with full __builtins__

The core tool executor runs exec() with the full built-in namespace accessible. LLM-generated tool implementations can read the filesystem, spawn processes, or make network calls.

LangChain — CVE-2025-68664

100% recall on a published CVE.

Rowan scanned 1,697 files and identified 58 instances of langchain.loads() with 100% recall and zero false positives. The vulnerable function deserializes untrusted data into arbitrary Python objects.

Cross-validated against the CVE ground truth.

The honest summary: Rowan is not a drop-in replacement for CodeQL on general Python. CodeQL is stronger on XXE and some deserialization patterns. Rowan’s edge is AI/ML-specific vulnerabilities and agentic analysis. For AI infrastructure teams, run both.

Known limitations

What we can’t do yet.

We maintain a public KNOWN_LIMITATIONS.md in the repository. Current gaps:

XXE / XML parsing rules — planned v0.2.1

Flask route-parameter taint — partial coverage only

No published precision/recall scorecard with confidence intervals yet

Cross-file taint for Java, JavaScript, Go, C# is within-file only

No scan sandbox yet — malicious repositories can affect the scanner process (containerization planned v0.3.0)
