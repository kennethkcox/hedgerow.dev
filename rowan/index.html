The limitation

Real vulnerabilities don’t stay in one file.

An SSRF that spans two files and an async class method won’t appear in a single-file regex scan. Rowan follows the full chain.

api/endpoints.py

# HTTP entry point
@router.get('/fetch')
async def fetch_resource(url: str):
  return await client.get(url)
# leaves unsanitized

clients/resource.py

# downstream — different file
class ResourceClient:
  async def get(self, url: str):
    return await httpx.get(url)
# SINK — no validation
[CRITICAL] Server-Side Request Forgery — confidence: 89%
Taint path: url param → client.get() → httpx.get()
            endpoints.py:3 → endpoints.py:4 → clients/resource.py:9
Note: No URL validation, scheme restriction, or allowlist detected

Coverage

What Rowan scans.

Cross-file taint analysis

Tracks user-controlled data from HTTP parameters, CLI arguments, environment variables, and file reads through function and import boundaries to dangerous sinks. A 4-hop chain from request.json to subprocess.run across three files is caught the same way as a direct call.

AI/ML-specific sink models

Serialization-based model loaders are treated as code execution primitives. trust_remote_code=True flags a trust boundary. apply_chat_template() is a prompt injection sink. Rowan models the full attack surface of SGLang, vLLM, LangChain, Letta, Gradio, and HuggingFace Transformers explicitly.

Static model file scanning

Analysis runs on the model files themselves, not just the Python code that loads them. Covers serialized checkpoint files, GGUF, SafeTensors, and Keras .h5 formats. Detects backdoored payloads and unsafe embedded code before the file is loaded.

Reachability-aware supply chain

Most SCA tools flag every CVE in every installed package. Rowan builds a call graph from your entry points and checks whether your code actually calls the vulnerable function. Only surfaces CVEs that are reachable. Suppresses the rest.

Prompt injection tracking

Taint flows from user-controlled input into LLM system prompts, RAG contexts, and Jinja2 template rendering are flagged as a distinct sink category. Tracked across multiple hops.

Framework CVE patterns

Dedicated rules for known vulnerability classes in the ML ecosystem: LangChain serialization injection, Keras importlib RCE, OmegaConf config injection, ONNX Hub path traversal, Gradio file read, and others.

Architecture

Four engines. One scan.

Each engine targets a different class of vulnerability. They run in parallel. Findings are deduplicated and confidence-scored before output.

CodeGuard

Taint analysis

Inter-procedural taint analysis with control flow graphs. Tracks user input across file and function boundaries. Python has full cross-file support. Java, JavaScript, Go, and C# use CFG-based taint within files.

NeuroScan

Pattern matching

125+ detection rules using tree-sitter AST parsing. Fast and language-agnostic. Catches hardcoded secrets, weak configurations, missing auth decorators, and AI-specific sink patterns without a full call graph.

DepGuard

Supply chain

Parses requirements.txt, poetry.lock, or pyproject.toml and queries the OSV database. Then checks reachability: only surfaces a CVE if your code actually calls the vulnerable function via a traceable import path.

Huntr Agent

Agentic hunt

An 8-stage autonomous pipeline: recon, dependency scan, hypothesis generation, deep-dive re-scan, exploit confirmation, live web probing (SSRF, path traversal, command injection), variant expansion, and full write-up with attacker steps. Works with Gemini, Claude, OpenAI, or MiniMax.

Output

Scored findings. Standard formats.

Confidence scoring

Every finding is scored 0–100% based on source clarity, taint path length, and whether known sanitizers were detected in the path. High-confidence findings are direct exploits. Lower scores identify patterns worth reviewing. Filter to 80%+ to separate signal from noise.

Setup

CLI and web API.

Runs locally with no infrastructure required. Also available as a FastAPI web service with project management, triage dashboard, and REST endpoints for CI integration.

# Scan a local project
rowan scan ./my-project

# Scan with SARIF output for CI
rowan scan ./my-project --format sarif --output findings.sarif

# Run the agentic hunt pipeline
rowan hunt https://github.com/org/repo --backend gemini

Changed-file scanning, PR security gates, and a GitHub Actions template are included. The free Gemini API tier is sufficient for most repositories.
