The problem

The tools you have weren’t built for AI.

CodeQL and Semgrep are excellent general-purpose scanners. But AI infrastructure introduces attack surfaces they don’t model.

torch.load() called on user-controlled paths. trust_remote_code=True in HuggingFace pipelines. Jinja2 template injection inside apply_chat_template(). eval() on LLM-generated strings.

These aren’t hypothetical. They’re patterns we found — confirmed, exploitable, in production frameworks used by thousands of teams.

Traditional SAST also stops at the file boundary. Real vulnerabilities don’t. An SSRF that spans four files and three abstraction layers won’t appear in a single-file regex scan. It requires tracking data flow across imports and function calls — and most tools don’t do that.

Capabilities

What Rowan does differently.

1

Cross-file taint analysis

Tracks data flow across function and file boundaries. Follows user input through imports, function arguments, and return values to reach dangerous sinks — wherever they live in your codebase.

2

AI/ML-specific sink models

Knows that torch.load(), pickle.loads(), dill.loads(), apply_chat_template(), and trust_remote_code=True are dangerous. Models the full attack surface of SGLang, vLLM, LangChain, Letta, Gradio, and HuggingFace Transformers.

3

Reachability-aware SCA

Doesn’t just list every CVE in your dependencies. Builds a call graph from your actual entry points and only surfaces vulnerabilities your code can reach. Eliminates the noise.

4

Confidence scoring

Every finding gets a 0–100% confidence score based on source clarity, path length, and sanitizers detected. Triage what matters first.

5

Autonomous hunt mode

An 8-stage agentic pipeline that generates hypotheses, traces exploit chains, and confirms exploitability — without manual effort. Built on LangGraph.

6

CI/CD native

Changed-file scanning, PR security gates, SARIF output, GitHub Actions template, webhook notifications. Fits into the workflow you already have.

Architecture

Four engines. One scan.

CodeGuard
Taint analysis

Inter-procedural taint analysis with control flow graphs. Tracks user input across file and function boundaries in Python, Java, JavaScript, Go, and C#.

NeuroScan
Pattern matching

125+ detection rules using tree-sitter AST matching. Fast, language-agnostic, no data-flow analysis required.

DepGuard
Supply chain

Supply chain scanning with reachability. Proves whether vulnerable dependency functions are reachable from your entry points before surfacing a finding.

Huntr Agent
Agentic analysis

LLM-powered agentic analysis. Generates attack hypotheses, traces multi-hop exploit chains, and confirms exploitability autonomously.
