What Rowan detects.

A full-stack scanner for AI/ML security — from model loading to prompt injection to supply chain vulnerabilities.

AI/ML vulnerabilities

The attack surface no one else models.

AI infrastructure creates a new class of vulnerabilities that general-purpose SAST tools weren’t designed to find. Rowan models these surfaces explicitly.

Unsafe model loading

Tracks user-controlled paths to pickle.load(), torch.load(), joblib.load(), dill.loads(), and cloudpickle.loads(). Detects when model file paths originate from HTTP parameters, environment variables, or CLI arguments without validation.

Static model file analysis

Scans .pkl, .pt, GGUF, SafeTensors, and Keras .h5 files for malicious payloads. Detects __reduce__ backdoors in serialized files. Flags Keras Lambda layers that execute arbitrary Python.

Trust boundary violations

Flags trust_remote_code=True in HuggingFace pipelines and hub_download() calls without pinned revisions. Models what happens when arbitrary code from the Hub runs in your inference process.

Prompt injection

Detects user input flowing into LLM system prompts, RAG contexts, and Jinja2 template rendering via apply_chat_template(). Tracks injection paths across multiple hops.

Code execution sinks

Tracks LLM-controlled strings to eval(), exec(), and compile(). Flags cases where the built-in namespace is accessible.

Framework CVEs

Detects known vulnerable patterns in LangChain (langchain.loads()), Keras importlib RCE, OmegaConf injection, ONNX Hub path traversal, and Gradio file read vulnerabilities.

Supply chain

CVEs that actually matter.

Most SCA tools give you a list of every CVE in every dependency. That list is mostly noise.

Rowan builds a call graph from your entry points — HTTP handlers, CLI commands, scheduled tasks — and traces whether the vulnerable function in a dependency is actually reachable from code that you call. If the vulnerable function is unreachable, Rowan suppresses the finding.

The result: a shorter list of vulnerabilities worth fixing, with evidence that your code can trigger them.

Confidence scoring

Not all findings are equal.

Every Rowan finding includes a confidence score from 0 to 100%, calculated from:

Source clarity

Is the taint origin clearly user-controlled?

Path length

How many hops between source and sink?

Sanitizers detected

Does the path pass through html.escape(), shlex.quote(), or other known mitigations?

Guard conditions

Are there boolean guards that would prevent the path from executing?

90–100%Critical — exploit path confirmed
70–89%High — strong taint evidence
50–69%Medium — plausible path, review recommended
30–49%Low — weak taint signal, manual verification required
<30%Informational
