How Thicket works

Four independent enforcement layers, active deception, and AI-native protections — each enforcing at a different point in the execution path.

Enforcement layers

Defense-in-depth from content to kernel.

Thicket installs four independent enforcement layers. An attacker who bypasses one immediately hits the next. Defeating the content scanner doesn't save them from the runtime trap. Evading the runtime won't save them from the kernel backstop.

1

Prompt & Content Scanning

Enforces before content reaches the model or triggers downstream execution

Every prompt, retrieved document, tool call, and model output is scanned before it reaches the model or triggers downstream execution. Injection patterns, jailbreaks, and malicious content embedded in documents — including obfuscated and encoded variants — are covered.

No content leaves your environment for scanning. All detection runs in-process against deterministic pattern sets.

2

Runtime Enforcement

Enforces at the execution boundary — inside the runtime, before operations complete

Even if an attacker bypasses content scanning, Thicket intercepts the resulting operation at the moment of execution — inside the runtime, before it completes.

This layer does not inspect text. It intercepts the actual execution triggers — network calls, file writes, and process spawning — blocking unauthorized operations at the execution boundary.

This is the layer cloud services cannot replicate. No amount of prompt obfuscation bypasses a hook on the operation itself.

3

OS Kernel Backstop (early access)

Linux kernel ≥ 5.10 with BPF LSM — no Kubernetes required

On compatible Linux hosts (kernel ≥ 5.10 with CONFIG_BPF_LSM), Thicket activates kernel-level enforcement that operates independently of the application runtime. Language-agnostic. Cannot be reversed from application code.

If an attacker manages to unhook application-level protections, any operations they attempt during that window are still caught here.

Application-level protections are also detected and re-applied automatically. The kernel backstop covers any window between tamper detection and reapplication.

4

eBPF DaemonSet (early access)

Kubernetes only — contact for early access

Extends enforcement node-wide — to every process on the host, including containers that do not import Thicket. Deployed as a Helm chart. Requires the eBPF daemon (early access — contact sales).

Where Layer 3 protects the instrumented process, Layer 4 protects the entire node. Any container on the node benefits from enforcement regardless of whether it imports the Thicket SDK.

Active Deception

Beyond blocking — Thicket actively traps attackers.

Detection and deception run alongside enforcement. When an attacker probes, they don't see a hard wall — they see plausible-looking responses, log every move, and eventually trigger a hard stop.

🍯 Honeypot Routes

Decoy endpoints catch reconnaissance scanners before they reach your application. Scanners receive plausible-looking synthetic responses. Every probe is logged with source IP, method, and path.

🪤 Agent Traps

If a compromised AI agent attempts to invoke a high-privilege operation it was never supposed to know about, Thicket treats that as an unambiguous adversarial signal and terminates the request — regardless of audit settings.

🎯 Canary Token Loop-Closing

Synthetic credentials served to probing requests are tracked. If those credentials appear in any subsequent payload — in any encoding — it triggers an immediate hard stop.

The attacker extracted fake data and tried to use it. Caught.
AI-Native Protections

Built for AI applications — not adapted from general-purpose tools.

Thicket includes protections specific to the AI stack that traditional application security tools do not cover.

🧮

Safe Expression Evaluation

AI applications increasingly need to evaluate user-supplied logic — custom rules, routing conditions, data filters, transformation expressions. Thicket provides a sandboxed evaluator that validates code structure before any execution occurs.

The dangerous built-in evaluation functions are never called. Safe for embedding in agent tools, flow builders, and notebook environments.

🖼️

Multi-Modal Security

Attackers don't only attack through text. Thicket scans image and media payloads for:

  • Injection instructions hidden in image metadata and embedded data channels
  • Malicious content embedded in SVG files
  • Suspicious payloads concealed using steganographic techniques
🤖

AI Model File Inspection

A model file is code. Before Thicket allows a model to be loaded, it inspects the file for:

  • Injection instructions embedded in model metadata and chat templates
  • Malicious content in system prompt fields
  • Structural anomalies in model architecture fields

Covers GGUF and ONNX formats.

🔗

Multi-Agent Trust & MCP Security

Agentic workflows introduce attack surfaces that single-model applications do not have:

  • Per-agent trust levels — cross-tier privilege escalation is blocked
  • Tool-result scanning — results from web searches and external tools are scanned before reaching the model
  • MCP tool-definition inspection — tool names and descriptions scanned at registration and call time
  • Denial-of-wallet protection — runaway agents are rate-limited before they exhaust API budgets
📦

AI Supply Chain Integrity

The dependency and model supply chain is an attack surface most AI security tools ignore:

  • Model integrity — cryptographic hash verification; any deviation blocks the load
  • Package typosquatting detection — scans your dependency list for lookalike AI package names
  • Model provenance — checks HuggingFace model metadata for supply chain risk signals
  • SBOM generation — CycloneDX-format Software Bill of Materials for compliance workflows
🔬

Built-in Code Scanner

Before you deploy, Thicket can scan your own Python and JavaScript codebase for dangerous source-to-sink flows — user-controlled inputs reaching command execution, deserialization, SSRF-prone HTTP calls, file access, and AI orchestration entry points.

Results export in SARIF format, compatible with GitHub Code Scanning, VS Code, and major CI pipelines.

🔄

Static Analysis Integration

Already running a SAST scanner? Connect its findings to the runtime engine.

Thicket can load results from your existing static analysis tooling and automatically elevate monitoring on code paths your scanner has already flagged. Static findings become active runtime signals.

👁️

Model Extraction Detection

Attackers can attempt to clone your model by sending high volumes of carefully crafted queries and collecting outputs. Thicket monitors query patterns per session and blocks extraction attempts before they yield meaningful results.

Output scanning also detects when model weight data appears in responses.

Audit & Compliance

Full audit trail, every enforcement action.

Structured audit log

Every enforcement action is logged — blocked category, request context, tenant ID, timestamp — and forwarded to your audit infrastructure. Syslog and Windows Event Log supported.

Audit-only mode

Records what would be blocked without enforcing. Evaluate policy impact before touching production — see exactly what would have been caught without disrupting your application.
