Thicket vs the alternatives

How runtime enforcement differs from cloud prompt scanners — and what to choose when.

Deterministic by design

Four reasons teams choose Thicket.

Deterministic enforcement you can reason about and audit.

No AI inside. Every blocking decision comes from a rule, a patch, or a kernel control — not a probability score. A block is a block: auditable, reproducible, explainable to your compliance team.

Zero external dependencies.

Thicket runs entirely in-process. No outbound calls, no API key, no cloud service. Suitable for air-gapped deployments, classified environments, and any situation where data sovereignty is a hard requirement.

Eliminates AI Act, NIS2, DORA, and GDPR data-sovereignty risk.

All scanning and enforcement happens on-premise, inside your process. No AI input or output ever reaches an external API — removing a significant compliance exposure for EU-regulated organisations.

Defense-in-depth — not a single layer.

Each enforcement layer operates independently. Bypassing one does not bypass the others.

Transparency

Known limitations.

We'd rather you hear these from us.

Prompt injection recall Not 100%. Novel patterns not matching known signatures may pass content scanning. Runtime enforcement and kernel controls remain active regardless.
Context detection Relies on call stack inspection. Unusual async execution patterns may occasionally misidentify context. A configurable override is available.
Rate limiting scope Per-process. In multi-worker deployments each worker maintains independent state.
In-memory patch tampering If a highly privileged attacker manages to unhook application-level protections, they are detected and re-applied automatically. Any operations attempted during that window are still caught by the OS kernel backstop.
SDK coverage Opt-in. LLM SDK protection must be explicitly activated.
FAQ

Common questions.

Does Thicket use AI or machine learning for detection?
No. Every enforcement decision is deterministic. No model is involved in any blocking decision.
Does it require internet access or an external service?
Zero external dependencies. Thicket is imported at startup. No outbound calls, no API key, no DNS resolution required.
Will it work in an air-gapped or classified environment?
Yes. Nothing leaves the process.
Does it support multi-tenant applications?
Yes. Per-tenant context isolation and rate limiting are built in.
What happens if a protection layer is tampered with?
Application-level protections are detected and re-applied automatically. Any operations attempted during that window are still covered by the OS kernel backstop.
Is it compatible with async frameworks like FastAPI?
Yes. All middleware is async-compatible.
What does the performance overhead look like?
Enforcement is in-process with no network round-trips. Audit-only mode lets you measure impact before enabling enforcement.
